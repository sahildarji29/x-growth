SDK Features

30+ modules across audio, browser automation, intelligence, state management, providers, plugins, memory, and turn coordination.

Browser Automation

Puppeteer CDP integration with self-healing selectors. Automatic login, Space join, speaker request, and audio injection via WebRTC.

Audio Pipeline

Real-time PCM capture, VAD with configurable silence detection, WAV encoding, and audio injection back into Space.

Intelligence Layer

Speaker identification, topic tracking, sentiment analysis, context windowing, and prompt building from conversation state.

Finite State Machine

Agent and team lifecycle driven by FSM with guards, actions, and history. No ad-hoc flags — all transitions are explicit.

Multi-Agent Teams

Deploy multiple AI agents with distinct personalities. Coordinated turn-taking, shared browser session, team state machine.

Middleware Pipeline

6 interception points: before/after STT, LLM, and TTS. Transform, filter, or abort processing at any stage.

Provider Router

Intelligent routing across multiple LLM providers. Automatic failover, cost optimization, and health monitoring.

Memory & Knowledge

Long-term memory with vector embeddings. RAG knowledge base for grounding responses in user-provided documents.

Decision Engine

Should-I-respond logic based on: directly addressed, is question, topic relevance, silence duration, last spoke.

Self-Healing Selectors

SelectorEngine tries CSS, text content, and aria attributes. Adapts when X changes their DOM — no manual maintenance.

Cost Tracking

Per-provider cost monitoring with USD estimates. Track tokens, requests, and spending across all AI services.

Plugin System

Extend agent behavior with plugins. Analytics, moderation, webhooks — load via PluginManager with lifecycle hooks.

Examples Gallery

12 runnable example projects covering everything from basic joins to multi-agent debates, Discord bridges, and scheduled Spaces.

🚀

Basic Join

Absolute minimum — join an X Space with an AI agent in ~15 lines. Listens via STT, responds with GPT-4o-mini + TTS.

const agent = new XSpaceAgent({ auth: { token, ct0 }, ai: { provider: 'openai' }, }) await agent.join('https://x.com/i/spaces/...')
beginner SDK
👥

Multi-Agent

Two AI agents (Bob & Alice) taking turns in one Space. Round-robin turn management with distinct voices.

const team = new AgentTeam({ agents: [bob, alice], turnStrategy: 'round-robin', }) await team.join(spaceUrl)
multi-agent teams
⚖️

Multi-Agent Debate

Two agents debate a topic live. Bull (OpenAI) vs Bear (Claude) — takes turns every 2s for up to 20 turns, then summarizes.

// Bull uses alloy voice, Bear uses echo // Concludes with debate summary const debate = new AgentTeam({ agents: [bull, bear], maxTurns: 20, })
debate multi-llm
🔮

Discord Bridge

Control the X Space agent from Discord. Commands: join, leave, speak, view transcriptions. Bridges two platforms.

// Discord commands: // !join <space-url> // !leave // !speak <message> // !transcriptions
discord bridge
🛠

Custom Provider

Use a local LLM (Ollama) or custom API. Runs LLM locally, only TTS uses cloud. Implement your own provider interface.

const agent = new XSpaceAgent({ ai: { provider: 'custom', endpoint: 'http://localhost:11434', } })
ollama custom
🔗

Middleware Pipeline

Demonstrates all 6 middleware hooks: noise filtering, language detection, safety redaction, and analytics at each stage.

agent.use('after:stt', filterNoise) agent.use('before:llm', injectContext) agent.use('after:llm', redactSafety) agent.use('after:tts', logAnalytics)
middleware hooks
📅

Scheduled Spaces

Join X Spaces on a cron schedule with auto-leave timers. Mon-Fri 9am crypto show, Fri 6pm AMA, Sun 3pm recap.

// Schedule: cron-based // Mon-Fri 9:00 → Crypto Show // Friday 18:00 → AMA Session // Sunday 15:00 → Weekly Recap
cron schedule

Transcription Logger

Listen-only agent. Saves timestamped transcripts to file. Never speaks — only records. Outputs to stdout + transcript.txt.

agent.on('transcription', ({ text, speaker }) => { appendFile('transcript.txt', `[${timestamp}] ${speaker}: ${text}\n`) })
listen-only logging
⚡

Express Integration

Embed agent as a sub-router in your existing Express app. Your routes + agent admin panel + API all on one server.

const app = express() app.use('/agent', agentRouter) // /agent/admin → Admin UI // /agent/api/status → API
express integration

Plugins

Three plugin examples: analytics tracking, content moderation, and webhook notifications. Shows the full plugin API.

agent.loadPlugin(new AnalyticsPlugin()) agent.loadPlugin(new ModerationPlugin()) agent.loadPlugin(new WebhookPlugin({ url }))
plugins extensibility

With Plugins

Complete agent setup with plugins loaded together. Shows how to register and compose multiple plugins into one agent.

const agent = createAgent({ plugins: [analytics, moderation, webhook], // All plugins active simultaneously })
complete production

Chrome Connect

Connect to an existing Chromium instance instead of launching a new browser. Uses connect mode instead of managed.

const agent = new XSpaceAgent({ browser: { mode: 'connect', wsEndpoint: 'ws://localhost:9222', } })
